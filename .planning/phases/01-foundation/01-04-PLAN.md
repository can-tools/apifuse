From .planning/phases/01-foundation/01-PATTERNS.md — exceptions.py pattern (verbatim):

Class: ErrorHandlingMiddleware
  - NOT BaseHTTPMiddleware — pure ASGI callable (see RESEARCH.md Pattern 5 rationale)
  - Imports: structlog, JSONResponse from starlette.responses, ASGIApp/Receive/Scope/Send from starlette.types

Constructor:
  __init__(self, app: ASGIApp, app_env: str = "development") -> None
    self.app = app
    self.is_dev = app_env == "development"

ASGI call:
  async def __call__(self, scope: Scope, receive: Receive, send: Send) -> None
    if scope["type"] != "http": pass through (no handling for websocket/lifespan)
    try: await self.app(scope, receive, send)
    except Exception as exc:
      status_code = getattr(exc, "status_code", 500)
      if self.is_dev: error_message = f"{type(exc).__name__}: {exc}"  (D-07: full detail in dev)
      else: error_message = "Internal server error" if status_code == 500 else str(exc)  (D-07: minimal in prod)
      log.error("apifuse_unhandled_exception", exc_type=type(exc).__name__, status_code=status_code, exc_info=exc)
      response = JSONResponse(status_code=status_code, content={"error": error_message, "status_code": status_code})  (D-08)
      await response(scope, receive, send)

Module-level logger:
  log = structlog.get_logger()

Decisions:
  D-07: dev→full detail, prod→minimal (500="Internal server error", others=str(exc))
  D-08: {"error": "<message>", "status_code": <N>} flat envelope

From app/core/config.py (Plan 02):
  apifuse_settings singleton available; ErrorHandlingMiddleware does NOT import it directly
  (app_env is passed as constructor argument by main.py to keep middleware testable)
