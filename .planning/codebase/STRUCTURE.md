# Codebase Structure

**Analysis Date:** 2026-05-12

## Current State vs. Planned State

This project is in early initialization. Only `main.py` (stub) exists as application code.

## Directory Purposes

**`app/api/`:**
- Purpose: Static FastAPI routers for fixed system endpoints
- Contains: Health checks, OpenAPI introspection routes, dev-only debug routes
- Key files: `health.py`, `openapi.py`, `debug.py`

**`app/core/`:**
- Purpose: Shared infrastructure — the foundation all other modules depend on
- Contains: HTTP client abstraction, provider registry, dynamic route generator, settings, logging, exceptions
- Key files: `client.py` (BaseProvider, ProviderRegistry), `config.py`, `dynamic_router.py`

**`app/models/`:**
- Purpose: Pydantic data models shared across layers
- Contains: `APIResponse`, `ErrorResponse`, dynamically generated request/response models from OpenAPI schema

**`app/providers/`:**
- Purpose: Concrete provider implementations; each knows how to call a category of external APIs
- Contains: `openapi_provider.py` (YAML-driven), `composite_provider.py` (aggregator), `provider_loader.py` (bootstrap), `debug/` (dev-only stubs)

**`config/providers/`:**
- Purpose: Declarative YAML definitions for each registered provider (type, base_url, auth, env_mapping)
- Contains: One `.yaml` file per provider; these drive `ProviderLoader` at startup
- Key pattern: Adding a new provider = adding a YAML file here, no Python changes needed

**`config/openapi/`:**
- Purpose: Fully resolved OpenAPI specs (all `$ref` expanded) that `DynamicRouterGenerator` reads to create routes
- Contains: `*-resolved.yaml` files; generated by `scripts/resolve_openapi.py`

**`docs/`:**
- Purpose: Human-readable project documentation (not auto-generated)
- Key files: `architecture.md` (system diagram), `plan.md` (phased roadmap)

**`scripts/`:**
- Purpose: Developer utility scripts for working with OpenAPI specs
- Contains: `resolve_openapi.py`, `validate_openapi.py`

**`tests/`:**
- Purpose: pytest test suite — unit tests for core modules, integration tests with mocked httpx
- Structure: mirrors `app/` layout with `test_*.py` naming

## Key File Locations

**Entry Points:**
- `main.py`: FastAPI application instance, lifespan handler, middleware stack, startup logic

**Configuration:**
- `pyproject.toml`: Project metadata, runtime and dev dependencies, Python version constraint
- `uv.lock`: Locked dependency graph (committed to VCS)
- `.python-version`: Pinned to `3.12`
- `config/providers/*.yaml`: Per-provider YAML configs (auth, base_url, env_mapping)
- `config/openapi/*.yaml`: Resolved OpenAPI specs consumed at startup

**Core Logic:**
- `app/core/client.py`: `BaseProvider` (abstract), `ProviderRegistry` (singleton), `CustomClient` (httpx wrapper)
- `app/core/dynamic_router.py`: `DynamicRouterGenerator` — parses OpenAPI YAML → FastAPI routes
- `app/core/config.py`: `Settings` class backed by pydantic-settings + `.env`
- `app/providers/provider_loader.py`: Scans `config/providers/`, instantiates and registers all providers

**Testing:**
- `tests/`: Root test directory
- `tests/conftest.py`: Shared pytest fixtures and app setup

## Naming Conventions

**Files:**
- Snake_case for Python modules: `openapi_provider.py`, `dynamic_router.py`
- Plural directories for collections: `providers/`, `models/`
- Suffix `_provider.py` for provider implementations
- Suffix `_loader.py` for bootstrap/discovery modules
- Config YAML files named after the provider: `docker.yaml`, `composite.yaml`
- Resolved OpenAPI specs: `{name}-resolved.yaml`

**Directories:**
- Lowercase snake_case: `app/`, `core/`, `providers/`, `config/`

**Python identifiers:**
- Classes: PascalCase (`BaseProvider`, `ProviderRegistry`, `DynamicRouterGenerator`)
- Functions/methods: snake_case (`execute_operation`, `_prepare_request`)
- Log event names: `apifuse_` prefix (e.g., `apifuse_provider_loaded`, `apifuse_route_registered`)
- Settings class: `Settings` (single class in `app/core/config.py`)

## Where to Add New Code

**New external API provider (YAML-driven):**
- Add provider definition: `config/providers/{name}.yaml`
- Add resolved OpenAPI spec: `config/openapi/{name}-resolved.yaml`
- No Python changes required for `openapi` provider type

**New custom provider (hand-coded):**
- Implementation: `app/providers/{name}_provider.py` (subclass `BaseProvider`)
- Register in: `config/providers/{name}.yaml` with `type: custom`
- Tests: `tests/providers/test_{name}_provider.py`

**New static endpoint (system/admin route):**
- Implementation: `app/api/{feature}.py` (new `APIRouter`)
- Mount in: `main.py` lifespan or app setup

**New shared utility:**
- Location: `app/core/{utility}.py`

**New Pydantic model:**
- Location: `app/models/{domain}.py`

**New test:**
- Unit test: `tests/{module}/test_{file}.py` mirroring `app/` structure
- Integration test: `tests/integration/test_{provider}.py`

**New dev script:**
- Location: `scripts/{purpose}.py`
